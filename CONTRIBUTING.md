# 贡献指南

感谢您对 ImageEdit Server 项目的关注！我们欢迎各种形式的贡献。

## 如何贡献

### 报告问题

如果您发现了 bug 或有功能建议，请：

1. 检查 [Issues](https://github.com/your-repo/issues) 中是否已有相关问题
2. 如果没有，请创建新的 Issue，包含：
   - 清晰的问题描述
   - 复现步骤
   - 预期行为和实际行为
   - 环境信息（Python 版本、操作系统等）

### 提交代码

1. **Fork 项目**

2. **创建功能分支**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **进行更改**
   - 遵循项目的代码风格
   - 添加必要的注释和文档
   - 确保代码通过测试

4. **提交更改**
   ```bash
   git commit -m "Add: 描述你的更改"
   ```
   
   提交信息格式：
   - `Add:` 新功能
   - `Fix:` Bug 修复
   - `Update:` 更新
   - `Refactor:` 重构
   - `Docs:` 文档更新

5. **推送到你的 Fork**
   ```bash
   git push origin feature/your-feature-name
   ```

6. **创建 Pull Request**
   - 提供清晰的 PR 描述
   - 说明更改的原因和影响
   - 关联相关的 Issue（如果有）

## 代码规范

### Python 代码风格

- 遵循 PEP 8 代码风格指南
- 使用 4 个空格缩进
- 行长度不超过 100 字符
- 函数和类需要添加文档字符串

### 提交信息规范

- 使用简洁明了的提交信息
- 第一行不超过 50 字符
- 详细说明可以写在后续行

示例：
```
Add: 支持批量图像处理功能

- 添加批量上传接口
- 实现并发处理机制
- 更新 API 文档
```

## 开发环境设置

1. Fork 并克隆项目
2. 创建虚拟环境
3. 安装依赖
4. 运行测试

详细步骤请参考 [SETUP.md](SETUP.md)

## 测试

在提交 PR 之前，请确保：

- [ ] 代码可以正常运行
- [ ] 没有引入新的错误
- [ ] 更新了相关文档
- [ ] 遵循了代码规范

## 许可证

通过贡献代码，您同意您的贡献将在与项目相同的许可证下发布。

## 问题？

如果您有任何问题，请：

- 查看 [README.md](README.md) 和 [SETUP.md](SETUP.md)
- 在 Issues 中搜索相关问题
- 创建新的 Issue 询问

感谢您的贡献！
